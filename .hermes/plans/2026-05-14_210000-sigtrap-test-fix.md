# Signal 5 (SIGTRAP) Test Infrastructure Crash — Fix Plan

## Problem
`swift test` exits with `signal 5` (SIGTRAP) and:
```
HummingbirdFluent/Fluent.swift:94: Fatal error: Unexpectedly found nil while unwrapping an Optional value
```
Preceded by repeated `PSQLError: too many clients already` messages — multiple test suites opening Fluent connections in parallel exhaust PostgreSQL's connection limit.

## Root Cause
- SPM test runner launches suites concurrently (default: N-1 workers on N cores)
- Each test file opens its own Fluent/Postgres connection
- No structured shutdown (`withTestFluent`) across all test files
- Connections pile up → Postgres hits `max_connections` → driver returns nil → force-unwrap crashes

## Immediate Workaround
```bash
swift test --parallel-workers 1
```
This serialises test execution. Trade-off: ~2x slower test runs, but zero crashes.

## Permanent Fix (Later)

The `withTestFluent` helper already exists in `Tests/AppTests/Support/FluentTestSupport.swift`.
It was created in the last session (HER-192) and migrated several test files already:
- `MemoryLineageTests` ✅
- `PhoneAuthFlowTests` ✅  
- `AccountDeletionTests` ✅

### Remaining files to migrate

Any test file that does **either** of these patterns needs migration:

```swift
// BAD — racy shutdown
let fluent = Fluent(...)
defer { Task { try? await fluent.shutdown() } }
```

```swift
// BAD — bare Fluent creation
let fluent = Fluent(logger: logger)
// ... no shutdown at all → crashes on deinit
```

To migrate, replace the above with:
```swift
try await withTestFluent { fluent in
    // test body
}
```

### Files still using the old pattern (grep to find):
```bash
grep -rl "Fluent(logger:" Tests/AppTests/ | grep -v FluentTestSupport
grep -rl "defer.*fluent.shutdown" Tests/AppTests/
```

Each file takes ~10 minutes to convert once the pattern is verified on the first file.
