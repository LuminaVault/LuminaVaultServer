# Output Template — External Content Ingestion

Use this template when generating the markdown file for an ingested article/tweet.

## Full Template

```markdown
# {title}

## Definition
{one_sentence_summary}

## Sources
- Original: {original_url}
- Mirror: {mirror_url_if_used}
- Date: {published_date}
- Author: @{handle} ({display_name})

## Summary
{bullet_point_summary_3_to_5_items}

## Key Points
- {key_point_1}
- {key_point_2}
- {key_point_3} (optional)

## Implications
{why_this_matters_to_vault_topics}

## Backlinks
- [[Related Page 1]]
- [[Related Page 2]]

---
**File imported from X**: {import_date}
**Original author**: @{handle} ({display_name})
**Vault topic**: #ai #devtools #... (comma-separated tags)
```

## Field Descriptions

| Field | Purpose | Source |
|-------|---------|--------|
| `title` | Article/tweet headline | Extracted from `<title>` or first H1 |
| `one_sentence_summary` | TL;DR essence | Condensed from first paragraph or generated summary |
| `original_url` | Canonical URL | User-provided link |
| `mirror_url_if_used` | If r.jina.ai or nitter was used | Optional, helps reproducibility |
| `published_date` | When article was posted | From meta tags or tweet `created_at` |
| `handle` | Author's X username | From URL path or tweet object |
| `display_name` | Author's display name | From tweet user object or meta tag |
| `bullet_point_summary` | 3–5 bullet summaries | Generated (or first 3 sentences) |
| `key_points` | Notable facts, quotes, numbers | Extracted salient details |
| `implications` | Why this matters for the vault | Context linking to existing topics |
| `backlinks` | Related wiki pages | Auto-detected via keyword matching |
| `import_date` | When added to vault | `datetime.now().strftime('%Y-%m-%d')` |
| `tags` | Vault topic tags | Derived from theme detection |

## Example Output

```markdown
# Sam Altman — ChatGPT subscriptions now work with OpenClaw

## Definition
OpenClaw, an alternative to Claude Code, now accepts ChatGPT Plus/Pro subscriptions for payment, giving users an alternative after Anthropic restricted some subscription usage.

## Sources
- Original: https://x.com/jeremynguyenphd/status/2050377302207524867
- Mirror: https://fixupx.com/jeremynguyenphd/status/2050377302207524867
- Date: 2026-05-02 10:15 UTC
- Author: @jeremynguyenphd (Jeremy Nguyen)

## Summary
- OpenClaw now authenticates with ChatGPT accounts
- Resolves disruption from Anthropic's subscription policy change
- Users can use OpenAI billing instead of Anthropic
- Part of shifting landscape of AI coding assistants

## Key Points
- Quote: "Direct from Sam Altman: You can use your ChatGPT subscription to power your OpenClaw."
- Affects users whose Claude Code access was restricted
- Alternatives: OpenCode (open-source), Hermes (agent platform)

## Implications
For Hermes users, this illustrates the risk of vendor lock-in for AI dev tools and the value of open alternatives.

## Backlinks
- [[Hermes Agent — The agent that grows with you]]
- [[OpenClaw as Claude Code alternative]]

---
**File imported from X**: 2026-05-02
**Original author**: @jeremynguyenphd (Jeremy Nguyen)
**Vault topic**: #AI #DevTools #OpenClaw #ChatGPT #Subscription
```

## Notes

- Keep `Definition` to **one sentence** (compressed TL;DR).
- `Summary` bullets should be **fragments**, not full sentences, where possible.
- `Implications` should tie back to existing vault topics; if none, write "No direct implications for current vault scope."
- `Backlinks` are auto-generated by searching for wiki page titles mentioned; manual edit later is fine.
- Date format: ISO-style `YYYY-MM-DD` for sorting; time optional in Sources section.
- Tags: lowercase, camelCase preserved, max 5 tags.
