# 横纵分析法深度研究Skill

## 使用场景

当用户想要深度、系统性研究一个产品、公司、概念、技术或人物时使用。特别适合：
- 新市场进入前竞争分析
- investment轮投资决策前
-产品战略规划
-技术方向选择
-概念理解和传播

## 核心方法

### 双轴分析框架

```
       纵轴（历时分析）
           ↓
    产品 ──技术──市场──用户──竞争──财经
  (产品)  (技术) (市场) (用户) (竞争) (财经)
       横轴（共时分析）
```

- **纵轴**：追踪从诞生到当下的完备演进路径，包括关键事件、转折点、版本迭代、市场变化、竞争动态等
- **横轴**：同时考察六大维度：
  - **产品**：功能、设计、用户体验、定位、 roadmap
  - **技术**：架构、算法、技术选型、创新点、技术债务
  - **市场**：市场规模、增长率、竞争格局、政策法规
  - **用户**：用户画像、需求痛点、使用场景、用户体验
  - **竞争**：竞争对手分析、竞争优势、竞争策略
  - **财经**：融资情况、收入模型、成本结构、盈利能力

## 输出结构

1. **基本信息**：名称、成立时间、创始人、所在地、行业等
2. **纵向演进**：时间线格式，关键里程碑
3. **横向分析**：
    - 产品分析
    - 技术分析
    - 市场分析
    - 用户分析
    - 竞争分析
    - 财经分析
4. **综合结论**：综合分析结果，给出投资建议、产品建议、技术建议等
5. **风险警告**：主要风险及应对建议

## 使用流程

1. 收集资料：收集关于研究对象的所有公开信息
2. 建立框架：建立双轴分析框架
3. 填充内容：填充六大维度的分析内容
4. 纵向追踪：建立时间线，追踪演进路径
5. 综合分析：综合六大维度的分析结果
6. 得出结论：根据分析结果得出结论

## 注意事项

- 保持客观性：避免主观偏见，基于事实进行分析
- 数据支撑：所有观点都要有数据或事实支撑
- 持续更新：随着时间推移，不断更新分析内容
- 动态调整：根据新的信息和变化，动态调整分析框架

## 输出格式

输出格式为Markdown，包含清晰的标题和列表，便于阅读和理解。
