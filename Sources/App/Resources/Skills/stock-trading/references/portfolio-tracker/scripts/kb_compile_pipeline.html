5
🏗️ Build Compressed Context
Hermes runs kb-compile strategy:
1. Cluster similar concepts (e.g., all "StockPlan" mentions)
2. Summarize each cluster into digest paragraphs
3. Build timeline of events (chronological narrative)
4. Extract patterns (recurring meetings, weekly goals, etc.)
5. Compress to ~20K tokens (Claude context window budget).
Result: A compressed_context.json that captures your life in ~15 minutes of reading.

🧠 Key Insights

  • Incremental compile: Only new/modified files since last run → 90% faster on subsequent runs
  • Compression is key: Raw vault texts might be 500K tokens. Compressed to 20K = 25× context reduction while preserving signal.
  • Entity resolution: "Fernando", "Drácarỳs", "fernando@fandemicapp.com" → same Person node. This is the magic.
  • Query routing: Simple queries ("latest meeting notes?") use vector search only. Complex queries ("What's my pattern on Mondays?") use graph traversal + LLM synthesis.
  • Caching layers: Query result → Redis (5 min), context → Redis (1h), embeddings → pgvector + HNSW (persistent).
